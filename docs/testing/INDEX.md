# 测试知识总索引

## 目录结构

### test-points/ — 测试点定义
按版本划分的测试点集合，共 155 个测试点。

- `Prism-测试点-索引.md` — 总索引：所有版本测试点汇总 + 状态统计
- `Prism-v0.1-MVP-测试点.md` — v0.1 MVP 阶段测试点
- `Prism-v0.2-测试点.md` — v0.2 阶段测试点
- `Prism-v0.3-测试点.md` — v0.3 阶段测试点
- `Prism-v1.x-测试点.md` — v1.x 阶段测试点

### checklists/ — 测试执行清单
按版本和 Phase 划分，12 份执行清单，记录每轮测试的具体执行状态。

- `test-checklist-v0.1-phase1~3.md` — v0.1 三个测试阶段
- `test-checklist-v0.2-phase4~6.md` — v0.2 三个测试阶段
- `test-checklist-v0.3-phase7~8.md` — v0.3 两个测试阶段
- `test-checklist-v0.4-phase9~11.md` — v0.4 三个测试阶段
- `test-checklist-v1.x-phase12.md` — v1.x 最终测试阶段

### e2e/ — 端到端测试
- `e2e-test-points.md` — E2E 测试点定义
- `e2e-test-results.md` — E2E 测试结果
- `e2e-retest-results.md` — E2E 回归测试结果
- `e2e-test-summary.md` — E2E 测试总结

### bugs/ — Bug 追踪与分析
- `bug-log.md` — Bug 日志（99+ 条，主记录）
- `bug-fix-log.md` — Bug 修复记录
- `rca-final.md` — 根因分析终版
- `pain-log.md` — 痛点日志（为改进积累真实痛点）
- `INDEX.md` — Bug 模式索引

### reports/ — 测试报告与状态
- `test-gap-analysis.md` — 测试覆盖 Gap 分析
- `progress-checkpoint.md` — 进度检查点
- `todo-from-testing.md` — 测试中发现的待办
- `env-status.md` — 环境状态记录
