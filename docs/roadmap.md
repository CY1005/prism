## 开发原则

1. **按功能纵切** — 一个功能从数据库到前端做完再做下一个
2. **每个版本都能跑** — 不做半成品，每周结束有可用的交付
3. **dogfooding** — 每两周用 Prism 录入真实数据验证体验
4. **卡住2小时就绕** — 记 pain-log，不死磕
5. **每天 git commit** — 保持可回退
6. **AI生成的代码先理解再用** — 不做代码搬运工
