## 一、核心功能清单（按版本）

### v0.1 MVP — 能录、能看、能搜、多人用

| # | 功能 | 一句话 | 复杂度 |

## 二、用户故事（按角色 × 功能域）

### 角色定义

| 角色 | 典型用户 | 核心诉求 |
|------|----------|----------|
| 项目管理员 | CY（项目创建者） | 建立项目知识体系，管理团队协作 |
| 编辑者 | 项目协作者 | 高效录入和维护功能项信息 |
| 查看者 | Mentor、+1 | 快速了解项目全貌和特定功能详情 |
| 平台管理员 | 运维角色 | 管理用户和平台配置 |

---

### A. 项目管理员的用户故事

#### A1 - 项目初始化

| ID | 用户故事 | 关联功能 |
|----|----------|----------|
| US-A1.1 | 作为项目管理员，我想创建一个新项目并选择"产品竞品分析"模板，这样系统自动配好维度和层级标签，我不用从零配置 | F2 |
| US-A1.2 | 作为项目管理员，我想在创建项目后修改维度的启用/禁用和排序，这样可以根据项目实际情况定制信息结构 | F2 |
| US-A1.3 | 作为项目管理员，我想修改层级标签名称（如"产品线→模块→功能项"改为"业务域→子系统→能力点"），这样标签能匹配项目的领域语言 | F2 |
| US-A1.4 | 作为项目管理员，我想为项目配置 AI Provider（Claude/Codex/Kimi），这样不同项目可以用不同的 AI 服务 | F2 |
| US-A1.5 | 作为项目管理员，我想通过 CSV 批量导入功能项，这样不用逐个手动创建，快速搭建项目骨架 | F11 |

## 三、功能依赖关系

```
F1 用户系统 ─────────────────────────────────┐
  │                                           │
F2 项目管理 ──┬── F3 模块树 ── F4 档案页 ──┬── F5 版本时间线
              │                              ├── F6 竞品参考
              │                              ├── F7 问题沉淀
              │                              └── F8 模块关系图
              │
              ├── F9 搜索（依赖 F4 的数据）
              ├── F10 全景图（依赖 F3 的树结构）
              └── F11 冷启动（依赖 F3/F4）

F12 对比矩阵 ←── F6 竞品参考数据
F13 需求分析 ←── F4 档案页 + F8 关系图 + AI 服务层
F14 行业动态 ←── F4 功能项关联
F15 数据流转 ←── 全局操作日志
F16 AI 快照  ←── F5 版本记录 + AI 服务层

F17 AI 辅助录入 ←── F4 + AI 服务层
F18 语义搜索    ←── F9 + pgvector
F19 导入导出    ←── F4 数据

F20 团队化 ←── F2 项目管理扩展
```

## 四、MVP 功能优先级排序（建议开发顺序）

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | F1 用户系统 | 一切的前提 |
| P0 | F2 项目管理（基础版） | 先能建项目 |
| P0 | F3 模块树 | 核心导航 |
| P0 | F4 档案页 | 核心页面，价值最大 |
| P1 | F5 版本时间线 | 档案页的子功能 |
| P1 | F6 竞品参考 | 档案页的子功能 |
| P1 | F7 问题沉淀 | 档案页的子功能 |
| P1 | F9 搜索 | 数据多了就需要 |
| P2 | F8 关系图 | 有价值但可视化开发量大 |
| P2 | F10 全景图 | 锦上添花 |
| P2 | F11 冷启动 | 有数据后才有意义 |
| P2 | F2 项目管理（完整版） | 成员管理、维度配置可后补 |
